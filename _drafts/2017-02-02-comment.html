Download a repo - 

from somewhere on the web.  We'll assume github, and since I'm working with mpiT right now, that's going to be the example.
mkdir ~/projects && cd ~/projects
git clone https://github.com/umhau/mpiT.git
We'll assume, from now on, that you're working from a github repo.

Modifying the repo

There's three steps in making a change to the repository.  First, you mess around and make some changes in your working directory.  Then you add the changes to the staging area, which tabulates the final difference between your new and old versions.  Finally, you push the staging area version of the changes to the 'head', which is the final check before uploading.  After pushing to the head, it can be uploaded.  It's complicated because it's designed to accommodate far more complex scenarios than this one. 

The directory the git clone command created is your working directory.  If you change a file in there, you can stage it and then upload it back to github (or wherever it came from).

Ok, now cd into the new directory.  
cd ./mpiT
Make a change - let's append something to the readme.
echo "This is super important stuff!" >> README.md
Cool.  Now we'll stage that change, and get it ready for uploading back to github.
git add README.md
Note you can also 'stage' everything at once, like this:
git add *
And add the 'stage' / 'index' to the 'head'.
git commit -m "Commit message"
Upload the 'head' to the server.
git push origin master
And if you haven't added the server yet, this is something that has to be done.
git remote add origin <server>
